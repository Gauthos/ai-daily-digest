🏷️ surveillance pricing, privacy, legislation, Maryland

## 3. 通过编译和测试不等于高质量软件：AI代码生成的局限

“A model that produces code which compiles and passes the tests it was given is not the same as a model that produces correct, secure, maintainable, well-architected software” — garymarcus.substack.com · 12 小时前 · ⭐ 23/30

AI模型能生成通过编译和单元测试的代码，但这远非衡量软件质量的标准。正确性、安全性、可维护性和良好架构往往被忽视，导致隐藏的技术债务和安全漏洞。许多开发者将编译和测试通过等同于“代码行得通”，而忽略了长期维护成本。Gary Marcus指出，这种误解可能导致生产环境中灾难性故障。应该建立更全面的软件质量评估框架，而非仅仅依赖表面指标。

🏷️ AI, code generation, software quality, LLM

## 5. Andrew Kelley谈识别LLM辅助代码的“数字气味”

Quoting Andrew Kelley — simonwillison.net · 1 天前 · ⭐ 20/30

Andrew Kelley指出，区分LLM辅助编写和纯人工编写的代码并不困难。人类犯错和LLM幻觉有本质不同，因此LLM辅助代码的痕迹很容易被察觉。长期使用agentic编码工具的人会带有独特的“数字气味”，自身难以察觉但他人很明显，如同吸烟者的体味。尽管无法100%捕获所有LLM辅助PR，但其模式特征使其难以隐藏。
